# Contributing

Thanks for contributing.

## How to contribute

- Fix typos, broken links, or unclear explanations.
- Improve examples for Spring Boot and AWS.
- Add diagrams, deployment steps, or troubleshooting notes.
- Add beginner-friendly explanations without unnecessary jargon.

## Basic workflow

1. Fork the repository.
2. Create a feature branch.
3. Make focused changes.
4. Test any code or commands you add.
5. Open a pull request with a clear summary.

## Writing guidelines

- Keep explanations step by step.
- Prefer practical examples over theory-only content.
- Mention AWS cost and security considerations where relevant.
- Do not commit secrets, credentials, or account IDs.
